# joojoo.ai.atelier

最終更新: 2026年6月7日

このリポジトリは、joojoo.ai.atelier のSNS画像・スライド画像制作フロー、カスタムGPT設計、事業計画を検証してきた作業場です。

## 目的

このカスタムGPTは、顧客ごとの世界観をもとに、SNS画像、スライド画像、講座資料、ワークショップ資料、告知資料などの制作を支援するためのものです。

単に画像を作るのではなく、目的、構成、1枚ごとの役割、必要な素材、生成と配置の切り分けを整理します。

## 目標

顧客が毎回ゼロからデザインを考えなくても、自分らしい資料を短い時間で作れる状態を目指します。

そのために、カスタムGPTは次のことを支援します。

- 制作物全体の構成を整理する
- SNS画像やスライド画像ごとの役割を決める
- AIに任せる部分と、固定して扱う素材を分ける
- 1枚ずつ、おさらいしてから生成または配置に進む
- 修正したい1枚だけを扱えるようにする

## Instructions

Instructionsには、カスタムGPT全体の役割、`画像生成` と `素材配置` のモード判定、1枚ずつ進めるための基本ルールだけを入れます。細かいデザイン方針や顧客別の世界観はKnowledgeに分け、判断できない場合だけ短く確認します。

## Knowledge

Knowledgeには、Instructionsに直接書き込まない情報を入れます。

**画像生成ルール**

SNS画像、スライド画像、表紙、章扉、問い、余韻ページなど、AIで1枚を作るときに使います。

**素材配置ルール**

提供された背景、写真、ワークシート、QRなどを配置するときに使います。

**顧客別デザインに関するもの**

顧客の世界観に合わせるために使います。デザインルールmd、参考画像、色や質感の資料などを含みます。顧客別資料のためGit管理しません。

## ファイル構成

今後のカスタムGPT運用で使う中心ファイルは、次の構成です。

```text
.
├── Instructions.md
├── 画像生成ルール.md
├── 素材配置ルール.md
└── customers/
    └── {customer_id}/
        ├── デザインルール.md
        └── 参考画像.*
```

`customers/` 配下は顧客別データのため、Git管理しません。

## 注意

`customers/` は顧客別データのためGit管理しません。

`private/` は事業計画などの非公開メモを置く場所です。Git管理しません。

`history/` は過去の試行錯誤、代表成果物、当時の関連mdやソースを残すための開発日誌です。顧客資料や生成物を含む可能性があるため、Git管理しません。
