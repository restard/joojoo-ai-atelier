# 無料管理画面 最小フロー

作成日：2026年5月29日

## 結論

無料管理画面のPhase 1は、AI構成生成を持たない。

役割は、ChatGPTまたはCustom GPTが作った `deck JSON` ファイルを安定してPPTX化するための、薄い操作盤に絞る。

最小フロー：

```text
JSONファイルを選ぶ
→ JSONを確認する
→ PPTXを生成する
→ ダウンロードURLを開く
```

6/8時点では、ログイン、顧客管理、素材アップロード、生成履歴、管理画面内AI生成は入れない。

## 対象ユーザー

### 主対象

- 谷内さん
- じゅじゅが代行操作するときの操作盤
- ChatGPTでdeck JSONまでは作れるが、APIやcurlは触りたくない人

### 対象外

- 自分でテンプレや素材を細かく管理したい上級者
- ブラウザ内でスライドを編集したい人
- AI生成まで管理画面内で完結したい有料会員

## 画面構成

実装URL:

```text

```

ローカル確認時:

http://127.0.0.1:8787/admin

### 1. JSONファイル選択画面

項目：

- JSONファイル選択ボタン
- 選択中のファイル名表示
- サンプルJSONファイルで試すボタン
- JSON確認ボタン
- 生成ボタン
- 補助用のJSON貼り付け欄

表示する補助情報：

- deck title
- slide count
- 使われている slide type
- 使われている color

主導線はファイル選択にする。

`Ctrl+A` / `Ctrl+C` / `Ctrl+V` が分からない人でも操作できるように、貼り付け欄は最初から大きく見せない。じゅじゅ代行や慣れている人のための予備導線として置く。

この画面では、JSONをファイルとして読み込めれば十分。スライド内容をフォーム分割して編集するUIは作らない。

### 2. 生成結果表示

成功時：

- `ok: true`
- ファイル名
- ファイルサイズ
- PPTXダウンロードリンク

失敗時：

- エラー種別
- 直す場所のヒント
- 元JSONを保持したまま再編集できる状態

## 最小バリデーション

管理画面側で見るもの：

- JSONとして読める
- トップレベルがobject
- `deck_title` がある
- `slides` がarray
- `slides.length > 0`
- 各slideに `type` がある
- 各slideに `content` がある

API側で最終判定するもの：

- 対応していない `type`
- 背景、画像、QRなど生成時に必要なファイル
- PPTX生成中の例外

管理画面は、完全な仕様チェックを目指さない。生成前に明らかなミスだけ止める。

## API接続

本線：

```text
POST https://gen-deck.onrender.com/generate
```

入力：

```json
{
  "deck_title": "sample_deck",
  "color": "terracotta",
  "slides": []
}
```

成功レスポンス：

```json
{
  "ok": true,
  "filename": "sample_deck.pptx",
  "relative_path": "dist/api/sample_deck.pptx",
  "download_url": "https://gen-deck.onrender.com/files/sample_deck.pptx",
  "size_bytes": 123456,
  "message": "PPTX generated and saved on the API server."
}
```

Custom GPT Action用のGET `generateDeckFromJsonQuery` は残すが、管理画面からはPOSTを使う。

## 6/8で確認するフロー

1. ChatGPTまたはCustom GPTでdeck JSONを作る
2. deck JSONをファイルとして保存する
3. 無料管理画面でJSONファイルを選ぶ
4. JSON確認で `deck_title` と枚数を見る
5. 生成ボタンを押す
6. PPTXダウンロードリンクを開く
7. PowerPointで開いて確認する
8. 修正したい場合は、ChatGPTにJSON修正を依頼して再生成する

- OpenAI APIキー入力
- 決済

## 入れるか迷ったが見送るもの

### 生成履歴

6/8では入れない。

理由：

- `download_url` が返れば確認できる
- 履歴保存にはユーザー識別や保存期間の判断が必要
- 無料版で保存を持つと、顧客資料の扱いが重くなる

### 素材アップロード

6/8では入れない。

理由：

- 現時点ではサーバー側にある固定素材だけを使う前提
- アップロードを入れると、ファイル保存、容量、削除、権限、画像パス変換が必要になる
- まずは見せデッキと既存素材でE2Eを確認する

### ブラウザ内プレビュー

6/8では入れない。

理由：

- PPTXをダウンロードしてPowerPointで見る方が、実利用に近い
- 画像/PPTX生成後のプレビューは別の実装コストがかかる
- 最小フローの成功条件は `download_url` 取得

## 無料版と有料版の切り分け

無料版：

- JSONファイル選択
- 補助用のdeck JSON貼り付け
- 簡易バリデーション
- PPTX生成
- ダウンロード

有料会員向け：

- 管理画面内AI構成生成
- ヒアリングテンプレ
- 生成履歴
- 再生成
- 会員向けテンプレ
- 素材管理

代行型：

- 顧客は原稿、写真、QR、特典情報を送る
- じゅじゅがChatGPTまたはCustom GPTでdeck JSONを作る
- じゅじゅが管理画面でJSONファイルを選び、PPTXを生成する
- 顧客にはPPTXファイルまたはダウンロードURLを渡す
- 本人操作を必須にしない

## 6/4タスクの決定事項

- 無料管理画面は、AI費用が発生しないJSONファイルアップロード型にする
- ここでのアップロードは、顧客素材や案件ファイルをサーバーに保存する機能ではない。ユーザーが選んだ `.json` ファイルをブラウザで読み込み、その内容を `POST /generate` に渡す最小機能を指す
- JSON貼り付けは、じゅじゅ代行や慣れている人向けの補助導線に下げる
- 6/8では、生成履歴と素材アップロードは入れない
- 管理画面からはRender APIの `POST /generate` を呼ぶ
- 成功条件は、PPTXファイルそのものを画面に埋め込むことではなく、`download_url` を開けること
- 修正フローは、画面上で細かく直すのではなく、ChatGPTにJSON修正を依頼して再生成する
- プランBでは、本人操作を必須にせず、じゅじゅ代行の操作盤としても使う

## 2026-05-29 実装・デプロイ記録

### 実装したもの

- `admin/index.html`: 静的な管理画面
- `gen_api.py`: `/admin`, `/admin.html`, `/sample-deck.json` を追加
- `gen_api.py`: `/` のレスポンスに `admin_url` を追加
- `README.md`: ローカル管理画面URLを追記

### 画面の機能

- `.json` ファイル選択
- サンプルJSON読み込み
- 補助用のJSON貼り付け
- 簡易バリデーション
  - JSONとして読める
  - トップレベルがobject
  - `deck_title` がある
  - `slides` がarray
  - `slides.length > 0`
  - 各slideに `type` と `content` がある
- `deck_title`, `slide count`, `slide type`, `color` の表示
- `POST /generate` によるPPTX生成
- `filename`, `size`, `download_url` の表示

- 白いノート/方眼紙の雰囲気
- 濃紺に近い黒の太い文字
- 境界線は黒ベース
- 影は使わない
- 水色は控えめなアクセントに限定
- 画面内の「6/8の流れ」パネルは削除
- タイトル下のマーカーは削除
- リード文下の横線は削除
- ファイル選択枠の中の方眼背景は削除
- 全体背景の方眼はかなり薄くする
