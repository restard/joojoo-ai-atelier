# 谷内さん環境E2E確認

作成日: 2026年5月29日
確認予定日: 2026年6月8日（月）

## 目的

谷内さん環境で、Custom GPTからgenDeck APIを呼び、PPTX生成結果の `download_url` を取得できるところまで確認する。

6/8時点の成功条件は、ChatGPT内にPPTXを直接添付することではなく、Render上でPPTXを生成して `download_url` を返せること。

## 本線

- Custom GPT: `genDeck Planner`
- API: `https://gen-deck.onrender.com`
- OpenAPI schema: `https://gen-deck.onrender.com/openapi.json`
- Action: `generateSmokeDeck`
- Action: `generateDeckFromJsonQuery`
- Authentication: `None`

## 事前確認

### 1. APIが起きている

```bash
curl -sS --max-time 60 https://gen-deck.onrender.com/health
```

期待値:

```json
{
  "ok": true
}
```

### 2. schemaのURLがRenderになっている

```bash
curl -sS --max-time 60 https://gen-deck.onrender.com/openapi.json
```

見るところ:

- `servers[0].url` が `https://gen-deck.onrender.com`
- `paths` は `/generate-smoke` と `/generate-json`
- operationId は `generateSmokeDeck` と `generateDeckFromJsonQuery`

### 3. GET版生成がcurlで通る

```bash
curl -sS --max-time 120 --get https://gen-deck.onrender.com/generate-json \
  --data-urlencode 'deck_json={"deck_title":"action_smoke_test","color":"terracotta","slides":[{"type":"statement","color":"deep_green","content":{"eyebrow":"Action test","lines":["接続テスト","PPTX生成"]}}]}'
```

期待値:

- `ok: true`
- `filename: action_smoke_test.pptx`
- `download_url` が `https://gen-deck.onrender.com/files/action_smoke_test.pptx`
- `size_bytes` が0より大きい

### 4. download_urlが開ける

```bash
curl -sS --max-time 60 -I https://gen-deck.onrender.com/files/action_smoke_test.pptx
```

期待値:

- HTTP 200
- `content-type: application/vnd.openxmlformats-officedocument.presentationml.presentation`
- `content-length` が0より大きい

## 失敗時の切り分け

### Render logsにリクエストがない

API内部ではなく、Custom GPT Actions側で止まっている可能性が高い。

見るところ:

- GPTのschemaが古くないか
- Available actionsにPOST版など不要なActionが混ざっていないか
- WorkspaceのAction/domain制限がないか
- GPT Previewではなく通常チャットで古いAction設定を掴んでいないか

### Render logsにリクエストがあり、APIが500を返す

API側またはPPTX生成側の問題。

見るところ:

- `deck_json` がJSON文字列として渡っているか
- `deck_title` と `slides` があるか
- Render logsのPython例外
- 背景/フォント/素材不足

### `download_url` は返るがChatGPT内で添付できない

6/8時点では成功扱いにする。

対応:

- `download_url` をブラウザで開く
- 必要ならサーバー側生成ファイルを手動共有する
- 次フェーズでファイル受け渡しのUIを作る
