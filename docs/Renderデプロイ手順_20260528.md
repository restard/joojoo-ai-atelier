# Renderデプロイ手順

作成日: 2026年5月28日

## 目的

ngrok無料ドメインでCustom GPT Actionsが `ClientResponseError` になるため、`gen_api.py` を通常のHTTPS公開URLに置いて再検証する。

## 追加した対応

- `Dockerfile`: Render Docker Web Service用
- `requirements.txt`: PPTX生成に必要なPython依存
- `render.yaml`: Render Blueprint用の最小設定
- `.dockerignore`: デプロイ不要な生成物と原本資料を除外
- `gen_api.py`: `PORT` / `HOST` / `PUBLIC_URL` 環境変数対応
- `gen_api.py`: `/openapi.json` が `X-Forwarded-Proto` と `Host` から公開URLを推定
- `gen_api.py`: 顧客素材なしの公開環境では安全なデモ背景を自動生成
- `slide_render.py`: macOSヒラギノがないLinux環境ではNoto CJKフォントへフォールバック

## ローカルDocker検証

2026-05-28にDocker Desktop 4.75.0で確認済み。`.dockerignore` で顧客素材と既存背景を除外した状態でも、デモ背景の自動生成でPPTX生成まで通る。

```bash
docker build -t gendeck-api:local .
docker run --rm -p 8791:8787 -e PORT=8787 -e HOST=0.0.0.0 gendeck-api:local
```

確認結果:

- `GET /health`: 成功
- `GET /openapi.json`: `X-Forwarded-Proto` と `Host` から公開URLを推定できる
- `GET /generate-smoke`: 成功。コンテナ内でPPTX生成まで完了
- Docker build contextは約40KBまで削減済み。顧客資料・背景・生成物は含めない

## Renderでの作成

1. GitHubにこのリポジトリをpushする
2. RenderでNew Web Serviceを作る
3. EnvironmentはDockerを選ぶ
4. Service nameは `gendeck-api` などにする
5. Free planでよい
6. デプロイ後、RenderのURLを開く

確認URL:

```text
https://{render-service}.onrender.com/health
https://{render-service}.onrender.com/openapi.json
https://{render-service}.onrender.com/generate-smoke
```

`/openapi.json` の `servers[0].url` がRenderのHTTPS URLになっていればOK。

## Custom GPT Actionsに貼るもの

Render公開後、次のURLをブラウザで開いて中身をコピーする。

```text
https://{render-service}.onrender.com/openapi.json
```

GPTのActions設定:

- Authentication: `None`
- Schema: 上記OpenAPI JSONを貼り付ける

## Previewで試す

まずは固定デッキで試す。

```text
generateSmokeDeckを呼んでください。
```

通ったら、次に `generateDeck` を試す。

```text

  "color": "terracotta",
```

## 成功条件

- Custom GPT Actionsから `generateSmokeDeck` または `generateDeck` が呼べる
- Render logsにリクエストが残る
- レスポンスに `ok: true` が返る
- `filename` / `relative_path` / `size_bytes` が返る

## それでも失敗する場合

Render logsにリクエストがない場合:

- Custom GPT Actions実行層
- GPT workspace/domain制限
- schema登録/モデル選択

Render logsにリクエストがあり、APIが500を返す場合:

- Render上のファイル不足
- フォント不足
- 背景プール不足
- PPTX生成例外

6/8のフォールバックは、Custom GPTが作ったdeck JSONをローカルAPIまたはcurlでPPTX化する。
