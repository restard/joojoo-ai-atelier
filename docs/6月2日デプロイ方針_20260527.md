# 6月2日デプロイ方針

作成日：2026年5月27日  
対象：6/8 E2E確認に向けたAPI公開方法

## 結論

6/8のE2E確認では、恒久デプロイではなく **ngrokによる一時公開** を第一候補にする。

理由:

- ローカルAPI `gen_api.py` はすでに動いている
- Custom GPT Actionから呼ぶにはHTTPSの外部URLが必要
- 6/8時点では本番運用ではなく、接続確認が目的
- Render/Fly.io等への恒久デプロイは、素材配置・ファイル保存・認証まで考える必要があり、今の目的に対して重い

## 6/2で決めること

### 1. Actionから呼べる場所

方針:

- ローカルAPIを `127.0.0.1:8787` で起動
- ngrokで `http://127.0.0.1:8787` をHTTPS公開
- 発行された `https://...ngrok-free.app` をCustom GPT Actionのserver URLに使う

### 2. 認証

6/8の確認では、API認証は入れない。

理由:

- URLを知っている人だけが使う一時公開に留める
- ngrokを起動している間だけ使う
- 認証実装より、まずAction接続可否を見る

注意:

- ngrok URLは不用意に共有しない
- E2E確認が終わったらngrokを停止する
- 商品化時は必ず認証方式を決める

### 3. ファイル受け渡し

6/8時点では、画像素材アップロードは扱わない。

方針:

- JSON内の画像パスは、サーバー側にすでに存在する相対パスを使う
- 代表6枚の顧客素材はGit管理に入れない。共有用には `specs/deck_sample.json` のようなサンプル化済みJSONだけを使う
- 顧客が新しい画像をアップロードする流れは次フェーズで検討する

### 4. 出力PPTX

APIはPPTXをHTTPレスポンスとして返す。

同時に、サーバー側にも以下へ保存される。

```text
dist/api/{deck_title}.pptx
```

6/8時点では、Custom GPTがファイルレスポンスを受け取れるかを確認する。難しい場合は、ローカルに保存されたPPTXを手動で渡すフォールバックにする。

## 手順

### 事前確認

ngrokが入っているか確認:

```bash
command -v ngrok
```

現時点では、Homebrew経由でngrokをインストール済み。

```text
ngrok version 3.39.5
```

初回起動にはngrokアカウントのauthtokenが必要。

起動テストでは次のエラーを確認した。

```text
ERR_NGROK_4018
Usage of ngrok requires a verified account and authtoken.
```

ユーザー作業:

1. ngrokアカウントを作成/ログインする
2. dashboardのauthtokenを取得する
3. 次のコマンドで設定する

```bash
ngrok config add-authtoken YOUR_AUTHTOKEN
```

2026-05-27追記:

- authtoken設定済み
- ngrok公開URLの発行を確認済み
- ngrok経由の `/health` / `/openapi.json` / `/generate` を確認済み
- `/generate` のレスポンスPPTXを `dist/api/prototype_0530_ngrok_download.pptx` として保存確認済み

### 起動手順

1. ngrokを起動する

```bash
ngrok http 8787
```

2. 表示されたHTTPS URLを控える

例:

```text
https://xxxx.ngrok-free.app
```

3. ローカルAPIを起動する

```bash
python3 gen_api.py --host 127.0.0.1 --port 8787 \
  --public-url https://xxxx.ngrok-free.app
```

4. OpenAPI schemaを確認する

```bash
curl http://127.0.0.1:8787/openapi.json
```

`servers[0].url` がngrok URLになっていればOK。

5. Custom GPT Actionから `/health` または `/generate` を呼ぶ

## 実装メモ

`gen_api.py` は `--public-url` を受け取れるようにしてある。これにより、Action用schemaを手作業で書き換えずに使える。

## フォールバック

ngrokが当日すぐ使えない場合:

- 6/8はローカルAPIのcurl実演で「JSON → API → PPTX」を見せる
- Custom GPT Action接続は障害として記録する
- 次候補としてRender/Fly.io等への簡易デプロイを検討する

## 6/2完了条件

- ngrok方針で進めることが決まっている
- 認証なし・素材固定・PPTXレスポンス返却の暫定方針が決まっている
- Custom GPT Actionから呼ぶための道筋が説明できる
- ngrokはインストール済み
- ngrok経由でPPTX生成APIを呼べる
